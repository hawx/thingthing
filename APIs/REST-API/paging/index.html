Paging

For requests on resources which generate a response the REST API will return an appropriate representation of the result in UTF-8 encoded XML. This section details the types of representation objects that describe the state of the resources being queried or manipulated. With each data representation, a list of expected Fields will be listed.

Paged Representations

When exposing certain large collections of data to the user, the REST API will return a subset of that information in a paged representation. Paged representations will have three defined XML attributes to give information on the scope of the current response and the overall size of the data collection.

For example, when the Inbox resource returns a collection of Message Header objects these are contained within a MessageHeaders XML element.

<messageheaders startindex="0" count="15" totalcount="239">
<messageheader …

In this example,

startindex="0"
The index of the first message in the returned data; the index of the first object of any collection is 0
count="15"
The number of messages returned.
totalcount="239"
The total number of messages in the Inbox. Further pages of data can be retrieved by making additional requests to the resource by shifting the startIndex value.

Paged collections have a maximum page size and requests for a larger ‘count’ value than is permitted will be reduced to the maximum allowed.
