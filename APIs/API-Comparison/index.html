Which API should I use?

Generally we would recommend you use the REST API, however, in some circumstances you may wish to use the FormPost or SOAP APIs.

REST API (recommended)

Our REST API is lightweight, easy to learn and built on established internet standards making it accessible from virtually any development language that can send and receive xml. In fact, this is the API our own web applications are built upon.

  • REST API quickstart guides
  • REST API reference

SOAP API

Our SOAP API supports our core messaging functionality and is ideal for quickly sending messages from any SOAP compatible client.

  • SOAP API reference

FormPost API

Our FormPost supports our core messaging functionality and allows you to send a message from pretty much anything that can access a url; be that a bespoke software application or third party product.
